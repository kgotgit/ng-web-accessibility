Role, Property, State and Tabindex Attributes

Role Attribute Element Usage
tree ul
  • Applied to the root ul tag to identify itself as tree widget
aria-labelledby="IDREF"
  • Refers to the ID Of the tree header. In the give example it refers to "Tree View"
treeitem li
  • Applied to the li element of ul
tabindex="-1"
  • Ensures the treeitem element is focusable but is not part of tab sequence. At any point of time only one treeitem element is included as part of tabsequence
tabindex="0"
  • Includes the treeitem element in the tab sequence
  • Only one treeitem in the tree has tabindex="0"
  • By default during the page load, the first treeitem of tree element is included in the tab sequence
  • When the user navigates through the treeitem elements, the element in focus will be included as part of tab sequence by setting tabindex="0" and other treeitem elements are set with tabindex="-1"
aria-expanded="false"
  • Applied to treeitem elements that are parent nodes i.e., they contain a ul with the group role
  • Indicates that the parent node is closed i.e., the child elements i.e., descendant elements are not visible which is syncrhonized with CSS attribute selector
aria-expanded="true"
  • Applied to treeitem elements that are parent nodes i.e., they contain a ul with the group role
  • Indicates that the parent node is open i.e., the child elements(i.e., descendant elements) are visible which is syncrhonized with CSS attribute selector
group ul
  • Identifies the ul element as the container of treeitem elements that form a branch of the tree
  • Browsers use the group to compute aria-level, aria-setsize and aria-posinet values for the nodes contained in the branch
